# Глосарій проєкту "SmartClock"
## Основні поняття предметної галузі

### Будильник (Alarm)
Підсистема, що дозволяє користувачу встановлювати звукові та візуальні сповіщення на точний час. Має атрибути: унікальний ідентифікатор, час спрацювання, статус (увімкнено/вимкнено), дні повторення та опис (назву).

### Таймер (Timer)
Інструмент зворотного відліку заданого проміжку часу. Підтримує функції запуску, призупинення (паузи), відновлення та має історію попередніх запусків для швидкого перезапуску.

### Секундомір (Stopwatch)
Інструмент для прямого вимірювання часу, що минув з моменту запуску, з можливістю фіксації проміжних результатів (кіл) та повного скидання результатів.

### Годинник (Clock)
Інструмент для перегляду поточного часу. Підтримую функцію часових поясів та 12-ти годинного режиму.

## Основні технології та фреймворки

### C++17
Сучасний стандарт мови програмування C++, який використовується як основна мова проєкту. Забезпечує високу продуктивність, сувору типізацію та ефективне управління пам'яттю.

### Qt6
Кросплатформний C++ фреймворк для розробки програмного забезпечення. У проєкті використовуються модулі `Core` (основна логіка), `Gui` та `Widgets` (побудова графічного інтерфейсу), а також `Multimedia` (відтворення звуків будильника та таймера).

### CMake
Система автоматизації збирання кросплатформного програмного забезпечення. Використовується для управління залежностями та розділення проєкту на незалежні модулі (`SmartClockLogic` та `SmartClockLib`).

### GoogleTest (GTest)
Фреймворк для написання та виконання модульних (unit) тестів на C++. Застосовується для ізольованого тестування бізнес-логіки (менеджерів та моделей) без залучення графічного інтерфейсу.

### JSON (через QJsonDocument)
Текстовий формат обміну даними. Використовується для локального збереження налаштувань, стану будильників, історії таймерів та результатів секундоміра.

### QSS (Qt Style Sheets)
Механізм стилізації графічних інтерфейсів Qt, подібний до CSS. Використовується для реалізації підтримки світлої та темної тем оформлення (`lightTheme.qss`, `darkTheme.qss`).

## Архітектурні компоненти

### Моделі (Models)
Компоненти, що відповідають за структуру даних та базову бізнес-логіку (незалежні від візуальної частини):
- `ClockModel` - відповідає за обчислення та форматування поточного часу.
- `StopwatchModel` - підрахунок мілісекунд, керування станами секундоміра та збереження кіл (laps).

### Контролери (Controllers)
Класи, що виконують роль посередника між інтерфейсом (View) та даними (Model):
- `ClockController`, `StopwatchController`, `AlarmController`, `TimerController` - обробляють дії користувача та взаємодіють з логікою без прямого втручання вікон у файлову систему чи моделі.

### Відображення (Views / Windows)
Графічні компоненти застосунку, які відповідають виключно за рендеринг UI та збір кліків:
- `MainWindow` - головне вікно-контейнер програми.
- `AlarmWindow`, `TimerWindow`, `StopwatchWindow`, `ClockWindow` - інтерфейси відповідних інструментів.

### Менеджери (Managers)
Центральні класи, що оркеструють виконання складних бізнес-процесів:
- `AlarmManager` - відстеження часу до спрацювання будильників та генерація подій.
- `TimerManager` - управління життєвим циклом таймерів зворотного відліку.
- `ThemeManager` - глобальне управління стилями та темами.

### Сховища (Storage Strategies)
Клас-стратегії, що реалізують абстрактні інтерфейси для збереження даних:
- `IAlarmStorage`, `ITimerStorage`, `IStopwatchStorage` - загальні інтерфейси доступу до даних.
- `JsonAlarmStorage`, `JsonTimerStorage`, `JsonStopwatchStorage` - конкретні реалізації для роботи з JSON-файлами.

## Патерни проєктування (Design Patterns)

### MVC (Model-View-Controller)
Архітектурний патерн, що розділяє систему на три частини (Модель, Відображення, Контролер). Використовується в підсистемах `Clock` та `Stopwatch` для повної ізоляції UI від бізнес-логіки та файлових операцій.

### Strategy (Стратегія)
Поведінковий патерн, який дозволяє виносити алгоритми в окремі класи. Використовується для механізмів збереження (Dependency Injection через `setStorage(...)`) та для виконання дій при спрацюванні будильника (`IAlarmAction` та `SoundAlarmAction`).

### Observer (Спостерігач)
Патерн для створення механізму підписки, який сповіщає об'єкти про зміни стану іншого об'єкта. Реалізований через механізм Qt Signals & Slots (наприклад, емісія сигналу `themeChanged` замість прямого впливу на `qApp`).

### Singleton (Одинак)
Твірний патерн, що гарантує існування лише одного екземпляра класу в програмі. Використаний для `ThemeManager` (потокобезпечна реалізація з приватним конструктором).

### Factory (Фабрика)
Твірний патерн, що делегує створення об'єктів окремому класу. `AlarmFactory` приховує в собі логіку створення `AlarmItemWidget` (включення/вимкнення прозорості, встановлення іконок).

## Функціональні компоненти

### Система Будильників (Alarm System)
Підсистема створення та управління користувацькими сповіщеннями:
- `RepeatMode` - enum клас для управління режимами повторення (OneTime, Daily, Workdays тощо).
- `Snooze` - функція відкладення активного будильника.
- `computeNextTrigger` - алгоритм визначення наступного спрацювання на основі днів тижня та поточного часу.

### Система Таймерів (Timer System)
Модуль зворотного відліку часу, який підтримує збереження історії (`HistoryTimerWindow`) та гнучке налаштування тривалості через `SettingsTimerDialog`.

### Секундомір (Stopwatch)
Інструмент вимірювання часу з функцією фіксації проміжних результатів. Містить `AnalogStopwatchDial` для візуалізації стрілочного циферблата.

## Розгортання та інфраструктура

### Clean Architecture Build (Архітектура збірки)
Фізичне розділення проєкту на рівні системи збірки CMake для зниження зв'язності (Coupling):
- `SmartClockLogic` - статична бібліотека "чистої" логіки.
- `SmartClockLib` - статична бібліотека UI-компонентів.
- `SmartClock` - фінальний виконуваний файл, який об'єднує бібліотеки.

### Continuous Integration (CI/CD)
Автоматизована система перевірки якості коду (GitHub Actions). Відповідає за збирання проєкту та запуск ізольованих юніт-тестів `GTest` при кожному коміті до репозиторію.
